基本教程：
	一.起步
	二.代码构成
		json:
			小程序配置
			app.json
				app.json 是当前小程序的全局配置，包括了小程序的所有页面路径、界面表现、网络超时时间、底部 tab 等。QuickStart 项目里边的 
			工具配置
			project.config.json
				通常大家在使用一个工具的时候，都会针对各自喜好做一些个性化配置，例如界面颜色、编译配置等等
			页面配置
			page.json
				page.json让开发者可以独立定义每个页面的一些属性
		wxml：
			wxml模板：WXML 充当的就是类似 HTML 的角色
			与html不同点：
				1.标签名字有点不一样
				2.多了一些 wx:if 这样的属性以及 {{ }} 这样的表达式
		wxss:
			WXSS 样式：WXSS 具有 CSS 大部分的特性，小程序在 WXSS 也做了一些扩充和修改
				1.新增了尺寸单位rpx
				2.提供了全局的样式(app.wxss)和局部样式(page.wxss)
				3.此外 WXSS 仅支持部分 CSS 选择器
		js：
			JS 脚本文件来处理用户的操作。
	
	三.小程序能力
		启动：执行app.js中的onLaunch()
		程序与页面：Page页面构造器，onLoad回调
		组件：在 WXML 写上对应的组件标签名字就可以把该组件显示在界面上，可以通过属性传递值给组件，可以通过 style 或者 class 来控制组件的外层样式
		API：自带wx:接口，需要注意的是：多数 API 的回调都是异步，你需要处理好代码逻辑的异步问题。
	四.发布准备
		用户身份：开发者，体验者...
		预览：开发者工具会自动打包当前项目，并上传小程序代码至微信的服务器，成功之后会在界面上显示一个二维码
		上传代码：上传代码是用于提交体验或者审核使用的。
			开发版：预览会上传一份打包代码到微信服务器，这个版本为体验版
			审核版：提交审核的版本
			线上版：用户能够使用的版本
