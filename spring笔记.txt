
一.依赖注入（DI）
	1.装配Bean
		自动装配
			java:@ComponentScan
			xml：<context:component-scan base-package=""/>
			@Autowired自动装配
		javaConfig
			@Configuration
		XML:
			xml配置文件
		混合配置：
			javaConfig引入配置文件
				引入javaConfig:@Import(Config.class)
				引入xml配置文件：@ImportResource("classpath:spring-config.xml")
			XML引入配置文件
				引入xml文件：<import resouce="spring-config.xml"
				引入javaConfig:<bean class="com.demo.config.Config"/>
	2.高级装配
		配置profile Bean
			在java中配置profile
				@Profile("dev")开发环境
				@Profile("prod")生产环境
				激活指定的profile，对应的Bean被创建。没有指定profil的Bean始终会被创建
			在XML中配置profile
				使用<beans profile="dev"></beans>
		激活profile
			设置spring.profile.active和spring.profile.default的值
			active的权限大于default
			多种方式设置
				作为DispatcherSevlet的初始化参数
				作为web应用的上下文参数
				作为JNDI条目
				作为环境变量
				作为jvm的系统属性
				在集成测试类上，使用@ActiveProfiles注解设置
			例如：在web.xml设置profile为dev
				<context-param>
					<param-name>spring.profiles.default</param-name>
					<param-value>dev</param-value>
				</context-param>
		条件化bean
			@Conditional 条件为真时创建bean
		处理自动装配的歧义性
			给定的接口有多个实现时，Spring无法确定选择哪个实现，会发生异常，
			解决办法：
				1.标识首选的bean
					java @Primary声明首选bean
					xml	 <bean primary="true"></bean>
				2.限定自动装配的bean
					java @Qualifier("iceCream")
					使用自定义注解限定
		bean的作用域
			单例singleton
			原型propotype
			会话session
			请求request
			
			选择作用域：@Scope，scope=""
		运行时值注入
			属性占位符
			Spring表达式语言
